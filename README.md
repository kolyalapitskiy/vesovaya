# Сайт производителя весов

SEO-сайт для производителя автомобильных, вагонных и промышленных весов с формой заявок, PostgreSQL и менеджерской админкой.

## Команды

```bash
npm run build
npm run start
```

После сборки готовый сайт находится в `public/`, а Node-сервер отдает страницы, API заявок и админку.

## PostgreSQL и заявки

1. Скопируйте переменные окружения:

```bash
cp .env.example .env
```

2. Запустите PostgreSQL:

```bash
docker compose up -d postgres
```

3. Установите зависимости и соберите сайт:

```bash
npm install
npm run build
npm run start
```

Форма отправляет заявки в `POST /api/leads`. Сервер сам создает таблицу `leads`, если ее еще нет.

## Админка

Админка доступна по адресу:

```text
http://localhost:3000/admin/
```

Логин и пароль задаются переменными `ADMIN_USER` и `ADMIN_PASSWORD`.

В админке менеджеры видят заявки, телефон клиента, страницу обращения, интересующий продукт, комментарий и могут менять статус:

- `new` - новая
- `in_progress` - в работе
- `called` - позвонили
- `closed` - закрыта

## Что заложено

- 140 коммерческих SEO-страниц: главная, категории, продуктовые, региональные и сервисные посадочные.
- Семантическое ядро в `src/data/seo-pages.mjs`.
- Категоризация продукции и ЧПУ URL.
- Хлебные крошки на страницах и `BreadcrumbList` JSON-LD.
- Микроразметка `Organization`, `WebSite`, `Product`, `Service`, `FAQPage`.
- `sitemap.xml`, `robots.txt`, `llms.txt`, `humans.txt`.
- Форма заявок на посадочных страницах.
- PostgreSQL-таблица заявок и закрытая админка для менеджеров.
- Готовые мета-теги, canonical, Open Graph.
- AEO-блоки: короткие ответы и FAQ для поисковых ответов.
- GEO-посадочные страницы по городам.

## HTTP/3

HTTP/3 включается на уровне хостинга или CDN, а не внутри HTML. В `deploy/Caddyfile` сайт проксируется на Node-сервер; Caddy включает HTTP/3 автоматически при HTTPS, если открыт UDP 443.
